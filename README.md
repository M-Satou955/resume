### 各プロジェクトの詳細

#### 2025/07 ~ 現在

**プロジェクト概要**:
toB 向け LMS システムの開発、リプレース

- **プロジェクト規模：**
  - 平均 4 ~ 6 人チームでのチーム開発
- **使用技術：**
  - バックエンド：PHP8.2、Laravel12
  - フロントエンド：Vue3.3 TypeScript, Vite, Scss, Tailwind CSS
  - インフラ：Docker、Ubuntu、EC2、Nginx
  - ツール：Git、Backlog、Slack
- **役割・担当箇所：**
  - 設計、コーディング、オンボーディング補助、顧客へのヒアリング
  - バックエンド API/バッチ処理の設計、実装 フロントエンド実装 CSS/SCSS 構造のリファクタリング

**課題・状況/取り組み/成果**

CSS 取り込み作業の非効率性の解消

- 1,5000 行ほどある CSS のため、デザイン変更時の影響範囲が不透明でデザイン取り込み作業が属人化しているという課題があり、機能単位での分割、SCSS 化を提案・実装。デザイン取り込み時間を約 75%削減（1 日 →1.5〜2 時間）

マスターデータ配信バッチ処理の実装

- テナント数増加に伴うマスターデータ配信処理で、N+1 問題によるパフォーマンス劣化リスクの懸念があったため、Eager Loading を活用し、N+1 問題を事前予防。現在も安定的にバッチ処理が稼働中

緊急バグ対応

- サービス停止リスクや商談のためバグへの迅速な対応体制が必要という課題があっため、PM/ビジネスダイドと密に連携し、仕様確認 → 原因特定 → 修正まで一貫して対応し、導入提案が成功。
  - パワーポイントや Google スライドでたたき台のスライドを作成することで、認識の齟齬を減らす、クローズドクエスチョンを意識するなどし、先方のコミュニケーションコストを減らすことを意識。

EC プラットフォーム管理システム開発の開発・保守

- **プロジェクト規模：**
  - 平均 4 ~ 6 人チームでのアジャイル開発
- **使用技術：**

  - バックエンド：Python 3.9, Django 4.2.13, Django REST Framework 3.14.0
  - フロントエンド：Vue 2, Element UI
  - インフラ：Amazon EC2, S3, RDS, CloudWatch, CodePipeline, SQS
  - DB：MySQL 8.2
  - 設計：OpenAPI,Swagger, ER 図, シーケンス図

- **役割・担当箇所：**
  - 設計、コーディング、保守、運用
  - バックエンド API 設計・実装、配送会社連携バッチの追加設計・ドキュメント化、OpenAPI による API 仕様書作成

**課題・状況/取り組み/成果**

会員ランクによる割引の不整合の解消

- 会員ランク割引機能において無効会員へ割引が適用されるという不具合を発見し、ディレクターにすぐ松竹梅で修正案を提示。API 側のロジックの修正とフロント側の表示制御を両面で実装し、不正な割引適用による売上損失の防止と顧客データの整合性を確保。

ドキュメント不足による対応コストの増加を解消

- 配送会社追加のたびに改修が必要だが、ドキュメントが不足しており対応コストが大きかったため、配送会社連携バッチの既存ロジックをドキュメント化し、追加手順を標準化。結果、新規配送会社追加時の開発リードタイム短縮を達成

#### 2024/04 ~ 2024/08

**プロジェクト概要**:
建設業の DX 化を支援する図面照査・ファイル管理プラットフォームの開発

- **プロジェクト規模：**
  - 平均 6 ~ 7 人チームでのチーム開発
- **使用技術：**

  - バックエンド：Python 3.10.8, FastAPI, SQLAlchemy 2, boto3
  - フロントエンド：TypeScript, Vue 3, Vuetify 3
  - 外部連携：Box Python SDK, AWS S3
  - インフラ：Amazon EC2, Docker
  - DB：MySQL 8.0
  - 設計：OpenAPI, ER 図, シーケンス図, ワイヤーフレーム

- **役割・担当箇所：**
  - 設計、コーディング
  - バックエンド API 設計・実装、開発環境の改善、パフォーマンス最適化、外部 SDK 連携の調査・実装

**課題・状況/取り組み/成果**
開発環境の非効率性の解消

- マイグレーションファイル 27 件が分散し、セットアップに約 15 分を要していたのでマイグレーションファイル 27 件を統合し DB 構造を整理、セットアップ時間を約 60%削減（15 分 →4 分）
- ホットリロード・デバッグ環境が未整備で、コード変更のたびに手動再起動が必要だったため、Docker のボリュームマウント設定によるホットリロード環境の構築とコンテナ内デバッグ環境を整備し、開発生産性の向上に成功

ダッシュボード画面のパフォーマンス劣化の対応

- レコード数増加に伴い、API 応答時間が平均 5 秒まで悪化し、ユーザー体験を著しく損なうレベルまで遅延。SQL の実行計画を分析し、Eager Loading、インデックス追加による段階的最適化＆API の責務の分割などを実行。最終的に API 応答時間を約 80%改善（5 秒 →1 秒）

Box Python SDK 連携の調査、ナレッジ化

- Box Python SDK の公式ドキュメントと実装に乖離があったため、SDK ソースコードを解析し API のエンドポイントや挙動を特定。調査結果をドキュメント化しチーム内で共有したことにより、5 営業日の遅延が解消し、他メンバーの実装速度向上に成功
